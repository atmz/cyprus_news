# Greek Podcast Script + Audio Generation

## Overview

Generate a ~5 minute Greek-language podcast from the daily Greek summary (`summary_el.txt`), enriched with details from linked articles. Output is a TTS-optimized script and an MP3 audio file.

## Input

- `summary_el.txt` — the Greek summary with article links (already generated by the pipeline)
- Linked article URLs extracted from the summary (fetched at podcast generation time)
- Cached article JSON files (titles + abstracts from `data/*.json`)

## Output

- `summaries/YYYY-MM-DD/txt/podcast_script_el.txt` — the script text
- `summaries/YYYY-MM-DD/media/podcast_el.mp3` — the audio file

## New module: `src/podcast.py`

### `generate_podcast_for_day(day: date) -> None`

1. Read `summary_el.txt` from the day's text folder
2. Parse markdown to extract:
   - Top stories (from "Κύριες Ειδήσεις" section)
   - All other section content (for brief roundup)
3. Extract article URLs from markdown links `[(Φ)](url)`, `[(SL)](url)`, `[(Π)](url)`
4. Fetch article text from URLs:
   - `requests` + BeautifulSoup first (simple extraction)
   - Playwright fallback for JS-rendered sites if needed later
   - 10s timeout per article, cap ~500 words per article
   - Graceful fallback to cached abstract from JSON if fetch fails
5. Match articles against cached JSON files for additional context
6. Send to GPT-4.1 with podcast prompt:
   - Summary text + fetched article content + cached abstracts
   - Prompt instructs: combine related top story bullets, add attribution ("Σύμφωνα με τον Πολίτη...", "Όπως αναφέρει η Σίγμα..."), write conversational script
7. Save script to `podcast_script_el.txt`
8. Send script to OpenAI TTS API → save as `podcast_el.mp3`

### Article fetching helper

`fetch_article_text(url: str) -> str | None`
- Uses `requests` with 10s timeout
- BeautifulSoup to extract article body (`<article>`, `<div class="article-body">`, etc.)
- Returns plain text, capped at ~500 words
- Returns None on failure (network error, timeout, JS-only site)

## Prompt design: `src/prompts/podcast_prompt_el.txt`

Based on the Marilou character from `example_prompt_el.txt`:
- Cypriot journalist persona, energetic, ~30% local humor after facts
- TTS-optimized: natural pauses with `---`, spell out acronyms, clear context for each story
- No section headers — flowing conversational text with transitions
- Structure:
  1. Brief intro with date
  2. 5-7 top stories (merged when related, enriched with article details + source attribution)
  3. Quick roundup of other notable stories from remaining sections
  4. Brief humorous closing
- Target: ~1200-1500 words (~5 minutes at TTS speaking rate)
- Only confirmed facts, no predictions or speculation

## Integration with `main.py`

- New `--podcast` CLI flag
- Runs after Greek summary generation (depends on `summary_el.txt` existing)
- Skips if `podcast_script_el.txt` and `podcast_el.mp3` already exist
- Wrapped in try/except so failure doesn't block the rest of the pipeline

## TTS

- OpenAI TTS API (`tts-1-hd` model for quality)
- Voice: to be selected (likely `nova` or `alloy` — user can configure)
- Output format: MP3
- Note: OpenAI TTS has a max input length; if script exceeds it, split into chunks and concatenate

## Caching

Same pattern as summaries: skip generation if output files already exist. Delete cached files to regenerate.
